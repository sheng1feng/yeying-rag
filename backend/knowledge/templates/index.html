知识库功能总览

1. 资产接入
自动连接 `warehouse`，支持浏览 `personal` / `apps`，并将个人上传仍然落到资产仓库。
2. 索引建库
按文件类型切块，生成 embedding，写入向量库，并维护导入任务明细与重试能力。
3. 记忆管理
支持长期记忆与短期记忆管理，并可在检索/对话后自动沉淀关键上下文。
4. 检索供给
为 bot / chat 提供检索结果和 retrieval-context，而不是直接替代对话服务。

API 接入文档

Step 1钱包 challenge / verify 登录 `knowledge`，获取 `knowledge JWT`。
Step 2按需调用 `/kbs/{kb_id}/search` 或 `/retrieval-context` 获取知识块。
Step 3bot / chat 生成最终回答后，调用 `/memory/ingest` 自动沉淀记忆。
Step 4后续会话继续复用 `retrieval-context`，系统会同时返回长短期记忆块。
交互式 OpenAPI：`/docs`。仓库文档：`docs/api-integration.md`。

核心接口

登录
`POST /auth/challenge` → `POST /auth/verify`
检索
`POST /kbs/{kb_id}/search`、`POST /retrieval-context`
自动记忆沉淀
`POST /memory/ingest`、`GET /memory/ingestions`
知识库管理
`/kbs`、`/tasks`、`/warehouse/*`、`/memory/*`

产品操作手册

1登录钱包后，系统自动尝试绑定 `warehouse personal`。
2在“资产仓库”中浏览或上传文件，再绑定到目标知识库。
3在“导入任务”中创建导入/重建/删除任务并跟踪任务明细。
4在“文档与切片”中核验 chunk，在“检索与上下文”中验证命中效果。
5在“记忆”中查看自动沉淀记录，并按需手工补充或删除记忆。
完整手册见 `docs/console-operations.md`。
