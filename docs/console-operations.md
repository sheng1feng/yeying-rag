# Knowledge 控制台操作手册

本文档面向测试、运营与交付同学，说明如何通过前台完成知识库联调与排查。

## 1. 登录与准备

1. 打开控制台首页
2. 点击“钱包登录”
3. 登录成功后，系统会自动尝试连接 `warehouse personal`
4. 在首页查看“系统就绪状态”

## 2. 创建知识库

1. 进入“知识库”
2. 输入名称、描述和基础检索参数
3. 点击“创建知识库”
4. 选中目标知识库作为当前工作对象
5. 若后续修改 `chunk_size` / `chunk_overlap` / `embedding_model`，控制台会自动为已有文档触发重建

## 3. 上传与绑定资产

1. 进入“资产仓库”
2. 在 `personal/uploads` 下上传文件
3. 上传成功后可直接点击“导入”或“定位”
4. 如果要长期纳入当前知识库，也可在“知识库”页绑定源路径

## 4. 导入与任务治理

1. 进入“导入任务”
2. 填写目标知识库与源路径
3. 创建导入 / 重建 / 删除任务
4. 在任务列表中查看状态
5. 点击“详情”查看 task item 成功、跳过、失败明细
6. 若失败，可使用“重试”

## 5. 核验文档与切片

1. 进入“文档与切片”
2. 查看导入后的文档列表
3. 选中文档查看 chunk 详情
4. 确认文件类型、chunk 数量与切片内容是否合理

## 6. 检索与自动记忆沉淀

1. 进入“检索与上下文”
2. 填写 `session_id`、`kb_ids` 与 `query`
3. 点击“生成上下文”
4. 将 bot / assistant 最终回答粘贴到回答输入框
5. 点击“沉淀本轮记忆”
6. 进入“记忆”查看长短期记忆与自动沉淀记录
7. 在控制台删除长短期记忆后，也会写入同一条记录流，便于回查

## 7. 运维排查

优先查看以下页面：

- “系统就绪状态”
- “运维状态”中的运行概览
- 存储与模型健康
- Worker 状态
- 最近失败任务

## 8. 当前产品边界

- `knowledge` 不回写 `warehouse`
- 当前不启用 rerank
- 自动记忆沉淀首版依赖 bot / chat 显式调用
- `warehouse` 仍是唯一资产中心
